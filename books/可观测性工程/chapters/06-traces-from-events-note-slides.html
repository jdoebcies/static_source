实践 · 硬编码探针
07 / 14

手动实现链路追踪只需三个步骤

Step 1

生成 Trace ID 与 Span ID

用 UUID 避免重复。将 span 属性和数据存储在字典中，序列化为 JSON 发送到后端。

Step 2

捕获时间戳与执行时长

记录请求开始和结束时间戳，差值即为持续时间，单位 ms。

Step 3

添加描述性字段

服务名 service_name 和工作类型 span 名 name，完成后通过 RPC 发送到链路追踪后端。

What

以键值对形式添加：执行主机 host、涉及用户 user_id、应用构建 ID build_id

核心总结
14 / 14
01

分布式链路追踪追踪单一请求跨多个服务的全过程。链路的价值在于清晰展示系统依赖关系，让上游延迟的根因不再被淹没在几十个服务中。

02

每个 span 至少需要五段数据：Trace ID、Span ID、Parent ID、时间戳、执行时长。通过 B3 或 W3C Trace Context 标准在 HTTP 请求头中跨服务传播上下文。

03

仅靠父子关系不够。加入主机、用户、build_id、运行时信息等自定义键值对，让每个 span 携带调试所需的完整上下文。

04

链路追踪不限于 RPC。非结构化日志可改造为结构化日志加链路字段；CPU 密集型操作、批任务、Lambda 流水线均可拆分为各自 span，构建瀑布视图。
